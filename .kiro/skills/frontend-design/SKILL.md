This skill guides creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. Implement real working code with exceptional attention to aesthetic details and creative choices.

The user provides frontend requirements: a component, page, application, or interface to build. They may include context about the purpose, audience, or technical constraints.

## Design Thinking

Before coding, understand the context and commit to a BOLD aesthetic direction:
- **Purpose**: This website is for Indie Coders and Content Creators looking for content creation tools. It should feel innovative and empowering.
- **Tone**: Minimal, sleek and professional with a touch of playfulness. Use white space and refined typography to convey sophistication, with pops of color for energy.
- **Audience**: Tech-savvy users who appreciate clean design but also want engaging visuals.
- **Constraints**: Must be responsive and performant. Use modern web standards.
- **Differentiation**: Avoid common design tropes. Instead of typical blue-and-white SaaS designs, opt for a unique color palette and unexpected typography choices.
- **Aesthetic Direction**: Decide on a clear visual style - for example, a refined minimalist aesthetic with generous white space, elegant typography, and subtle animations to create a sense of calm professionalism.
- **Design Principles**: Prioritize clarity, usability, and visual hierarchy. Use contrast, scale, and spacing to guide the user's eye and create a memorable experience. Make sure the whole design feels cohesive and intentional. Dont create different design styles in different parts of the same project. Keep the design language consistent throughout.

**CRITICAL**: Choose a clear conceptual direction and execute it with precision. Bold maximalism and refined minimalism both work - the key is intentionality, not intensity.

Then implement working code (HTML/CSS/JS, React, Vue, etc.) that is:
- Production-grade and functional
- Visually striking and memorable
- Cohesive with a clear aesthetic point-of-view
- Meticulously refined in every detail

## Frontend Aesthetics Guidelines

Focus on:
- **Typography**: Choose fonts that are beautiful, unique, and interesting. Avoid generic fonts like Arial; opt instead for distinctive choices that elevate the frontend's aesthetics; unexpected, characterful font choices. Pair a distinctive display font with a refined body font.
- **Color & Theme**: Commit to a cohesive aesthetic. Use CSS variables for consistency. Dominant colors with sharp accents outperform timid, evenly-distributed palettes. Use calm and light colors rather than dark themes unless the context demands it. Avoid overused color schemes (e.g., purple gradients on white backgrounds). Keep the UI visually minimal and avoid clutter.
- **Motion**: Use animations for effects and micro-interactions, not exceeding 300ms. Prioritize Motion library for React when available. Focus on high-impact moments: one well-orchestrated page load with staggered reveals (animation-delay) creates more delight than scattered micro-interactions. Use scroll-triggering and hover states that surprise. Do not overuse animations; every motion should have a purpose. Never apply animation on cards, list items or other repetitive elements. 
- **Spatial Composition**: Unexpected layouts. Asymmetry. Overlap. Diagonal flow. Grid-breaking elements. Generous negative space OR controlled density.
- **Backgrounds & Visual Details**: Create atmosphere and depth rather than defaulting to solid colors. Add contextual effects and textures that match the overall aesthetic. Do not apply generic background patterns or gradients.

NEVER use generic AI-generated aesthetics like overused font families (Inter, Roboto, Arial, system fonts), cliched color schemes (particularly purple gradients on white backgrounds), predictable layouts and component patterns, and cookie-cutter design that lacks context-specific character.

Interpret creatively and make unexpected choices that feel genuinely designed for the context. No design should be the same. Vary between light and dark themes, different fonts, different aesthetics. NEVER converge on common choices (Space Grotesk, for example) across generations.

**IMPORTANT**: Match implementation complexity to the aesthetic vision. Maximalist designs need elaborate code with extensive animations and effects. Minimalist or refined designs need restraint, precision, and careful attention to spacing, typography, and subtle details. Elegance comes from executing the vision well.

Remember: Gemini is capable of extraordinary creative work. Don't hold back, show what can truly be created when thinking outside the box and committing fully to a distinctive vision.
